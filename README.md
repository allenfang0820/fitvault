脉图（MaiTu）诞生于一个再普通不过的念头：能不能让运动数据变得更有价值？

过去一个月里，大叔利用业余时间和各种碎片时间，消耗了大量 AI Token，从零开始独立完成了这个项目。作为一名运动手表爱好者，我一直在使用 Garmin、COROS 等设备记录跑步、徒步、骑行等户外活动，但越来越觉得，传统运动平台展示的数据虽然丰富，却很难真正帮助普通用户理解自己每一次运动背后的意义。

于是，脉图诞生了。

脉图是一款轻量级、本地优先的运动数据分析平台。它能够解析运动手表产生的 FIT 文件，深入挖掘个人运动数据和户外轨迹信息，并通过接入 AI 智能体，对训练表现、身体状态、运动趋势以及轨迹故事进行多维度洞察。

与传统运动平台不同，脉图更关注“数据背后的故事”。它不仅告诉你跑了多远、爬升了多少，更希望帮助你理解：

- 为什么这次状态特别好？
- 哪个阶段出现了明显疲劳？
- 这条轨迹经历了怎样的环境变化？
- 长期训练趋势正在发生什么变化？
- AI 能从这些数据中发现哪些你自己没有注意到的细节？

这是一个由兴趣驱动的独立项目，也是一次关于运动数据与 AI 融合的探索实验。

希望脉图能够让每一次运动记录，都不仅仅是一串数字，而成为值得回味和思考的个人故事。

## 一、软件简介

脉图是一款本地优先的 AI 运动数据分析工具。它以 FIT 文件为核心数据来源，帮助用户导入、整理和复盘跑步、骑行、徒步、越野等运动记录，并通过可视化图表和 AI 解读，把原始运动数据转化为可理解的训练反馈。

脉图是一款免费、本地运行的运动数据分析工具，不会上传任何运动数据到云端。

脉图不是社交运动平台，也不是云端数据中台。它更像一个运行在本地的运动数据助手：负责保存可信运动档案、展示轨迹和指标、生成复盘洞察，并为 AI 助手提供稳定、可解释的运动上下文。

## 二、主要功能

### 1. 个人运动数据

个人运动数据页用于查看整体运动档案和历史活动记录。

主要能力包括：

- 查看 Garmin 用户画像、同步状态和基础运动信息。
- 按运动类型、标题关键词筛选历史活动。
- 分页浏览活动列表，查看距离、时长、爬升、心率、配速、功率等核心指标。
- 打开单条活动详情，进入概览或复盘分析。
- 删除不需要的运动记录。

### 2. FIT 数据导入与同步

脉图支持两种主要数据来源：

- 本地 FIT 文件：适合从手表、运动平台或备份目录导出的活动文件。
- Garmin 活动同步：佳明用户可在配置完成后按日期范围同步活动。

导入后，系统会解析 FIT 文件中的运动类型、时间、距离、轨迹、心率、配速、功率、海拔、圈数据等字段，并写入本地数据库。活动数据以本地 FIT 文件和后端解析结果为准，不依赖前端临时推算。

### 3. 活动详情

活动详情页用于快速了解单次运动。

你可以查看：

- 活动标题、时间、地点和运动类型。
- 距离、时长、爬升、消耗、平均心率、最大心率等核心指标。
- 真实比例轨迹缩略图。
- 圈数据、训练效果、环境挑战、路线难度等扩展指标。
- 轨迹缩略图入口，点击后可跳转到轨迹分析工具进行 3D 查看。

### 4. 运动复盘

复盘页面用于分析一次活动中身体状态和运动表现的变化。

它重点回答：

- 本次活动整体压力如何？
- 哪些阶段出现效率下降、心率漂移或疲劳累积？
- 是否存在能量断档、掉速、掉功率等线索？
- 哪些环境或路线因素影响了表现？
- 下一次类似活动应该如何调整节奏、补给或训练安排？

复盘数据来自后端权威快照，前端只负责展示，不自行补算事实字段。AI 洞察也只基于本次复盘快照生成，用于解释和建议，不会修改原始数据。

### 5. 轨迹分析工具

轨迹分析工具用于查看和分析 GPX/FIT 路线。

主要能力包括：

- 导入 GPX 轨迹文件。
- 在 3D 地球视图中查看路线。
- 查看海拔剖面、距离变化和路线统计。
- 在地图和剖面图之间联动定位。
- 导出 GPX 文件。
- 生成路线报告和活动建议。

对于 GPX 文件，脉图可以解析轨迹、距离、爬升和位置等路线信息；但 GPX 通常不包含完整的心率、功率、训练效果等设备字段。如需完整运动指标，建议优先使用 FIT 文件。

### 6. 雷达图与滚动指标

雷达图用于观察近期训练状态和关键能力维度。

系统会基于最近活动和滚动窗口计算指标，例如：

- 训练负荷趋势。
- HRV 或恢复相关基线。
- 心率漂移。
- 爬升能力。
- 阈值心率。
- 无氧峰值或强度表现。

雷达图 AI 洞察用于解释这些维度的变化，帮助你判断近期训练是偏恢复、偏积累，还是存在疲劳或强度失衡的迹象。

### 7. AI 功能

脉图的 AI 功能主要用于解释和建议，不作为事实来源。

常用入口包括：

- AI 洞察：针对单次活动或雷达图生成结构化解读。
- 复盘 AI 解读：基于本次运动复盘快照总结疲劳、效率、事件和建议。
- 活动建议：基于当前路线事实和用户手动填写的计划信息，生成补给、天气检查、装备和体力安排建议。
- 全局 AI 助手：根据当前运动画像和已导入数据回答训练相关问题。

AI 生成内容仅供训练参考。它不会改写 FIT 文件、不会写入原始指标，也不会把 AI 判断当作数据库事实。

### 8. 系统配置

配置页用于完成首次使用准备。

你可以在这里设置：

- 大模型供应商。
- API 地址。
- 模型名称。
- Token 或鉴权信息。
- 手表品牌。
- 本地 FIT 文件存储目录状态。
- 网关连接测试。
- 存储规范同步。

配置完成后，脉图会在后续启动时进行静默验证。若配置页出现连接异常提示，需要重新测试网关或检查 API 信息。

## 三、快速上手

### 第一步：完成基础配置

1. 打开脉图。
2. 进入「配置」页。
3. 填写大模型 API 地址、模型和 Token。
4. 点击「测试连接」。
5. 选择手表品牌。
6. 点击「告诉 AI 助手存储规范」完成初始化。

配置成功后，后续一般不需要重复操作。

### OpenClaw 配置说明

OpenClaw 是脉图里非常重要的本地网关。可以把它理解成脉图和本地 AI 助手、MCP 技能、Garmin 活动下载能力之间的连接桥。配置好 OpenClaw 后，脉图才能稳定使用 AI 洞察、活动建议、复盘解读，以及按日期同步佳明活动 FIT 文件。

如果你使用佳明，需要在本地 OpenClaw 或 QClaw 中安装 `garmin-stats` skill；如果你使用高驰，需要安装 `coros-stats` skill。这两个 skill 用于从佳明/高驰账户获取用户画像数据；其中 `garmin-stats` 还支持佳明活动 FIT 文件同步。下载对应 zip 后，在 OpenClaw / QClaw 中使用 Skill 安装/导入功能选择该 zip 即可。

Skill 下载：

- 佳明用户：[garmin-stats.zip](../skills/garmin-stats.zip)
- 高驰用户：[coros-stats.zip](../skills/coros-stats.zip)

安装后按 OpenClaw 显示的 skill 说明完成依赖安装和账号授权，再回到脉图执行「测试连接」和「告诉 AI 助手存储规范」。

使用注意事项：

- 两个 skill 都需要用户在本机完成账号授权。脉图不会询问、保存或记录佳明/高驰账号密码。
- `garmin-stats` 首次使用需要登录 Garmin 账号并保存本地 OAuth 授权；如果授权失效，需要在 OpenClaw 中重新执行登录流程。
- `garmin-stats` 的睡眠、HRV、静息心率、体重、VO2max、阈值和比赛预测会实时获取；累计里程、最长距离等活动历史字段使用活动缓存，缓存超过 7 天会自动刷新。
- `coros-stats` 需要使用高驰 MCP 授权账号；Training Hub 数据还需要在本机 Chrome 中登录 `https://t.coros.com/admin/views/dash-board`。
- `coros-stats` 的 Training Hub 通道可能会唤起或连接一个带 DevTools 端口的 Chrome，用于读取已登录的 Training Hub 页面。请在浏览器中自行完成登录，不要把密码发给 AI 助手。
- 如果 Chrome 被关闭、Training Hub 登录态过期，或高驰页面结构变化，`coros-stats` 会尽量退回 MCP 可用字段；PB、最大心率、静息心率等 Training Hub 字段可能暂时返回 `null`。
- 首次授权、首次缓存生成或活动历史刷新可能比普通静默同步更慢；后续同步会优先复用有效授权和缓存。
- 安装或更新 skill 后，建议重启 OpenClaw / QClaw，使新 skill 和依赖被重新加载。

在「配置」页中，OpenClaw 相关字段通常这样理解：

- 服务商：选择「本地 + MCP（OpenClaw / QClaw）」。
- 自定义 API 地址：填写 OpenClaw 提供的网关地址，例如本机服务地址或你实际部署的网关地址。
- 模型：选择或填写 OpenClaw 网关可调用的模型名称。
- Agent ID：如果 OpenClaw 给你分配了专用 Agent ID，就填入这里；没有则可以留空。
- Gateway Token：如果 OpenClaw 网关开启了鉴权，就填写 Token；如果你的本地网关没有鉴权，可以留空。

配置时建议按这个顺序操作：

1. 先确认 OpenClaw / QClaw 已启动，并且已按设备类型安装对应 skill：佳明安装 `garmin-stats`，高驰安装 `coros-stats`。
2. 在脉图「配置」页填写 OpenClaw API 地址、模型、Agent ID 和 Token。
3. 点击「测试连接」。测试成功后，脉图会保存这组网关配置。
4. 选择手表品牌。如果你要使用按日期同步 Garmin 活动，请选择「佳明」。
5. 点击「告诉 AI 助手存储规范」。这一步会把 FIT 文件固定保存目录告诉 OpenClaw，确保它下载活动后把 FIT 文件放到脉图能扫描的目录。

「测试连接」和「告诉 AI 助手存储规范」不是同一件事：前者确认网关能连通，后者确认 OpenClaw 知道 FIT 文件应该下载、解压并保存到哪里。Garmin 同步依赖这两步都完成；如果只测试连接、不同步存储规范，OpenClaw 可能下载成功但脉图找不到文件。

### 第二步：导入运动数据

方式一：导入本地 FIT 文件

1. 进入「个人运动数据」页。
2. 点击「导入本地 FIT 文件」。
3. 选择 FIT 文件或 ZIP 压缩包。
4. 等待系统解析并刷新活动列表。

方式二：同步 Garmin 活动

1. 确认手表品牌选择为佳明。
2. 确认 OpenClaw 网关连接正常。
3. 确认配置页已完成「告诉 AI 助手存储规范」。
4. 在「个人运动数据」页选择开始日期和结束日期。
5. 点击「同步活动」。
6. 等待 OpenClaw 下载 FIT 文件，脉图会随后同步本地 FIT 数据并刷新活动列表。

方式三：导入 GPX 轨迹

1. 进入「轨迹分析工具」页。
2. 点击「导入 GPX」。
3. 选择 GPX 文件。
4. 查看 3D 轨迹、海拔剖面和路线报告。

### 第三步：查看活动详情

1. 在「个人运动数据」页找到一条活动。
2. 点击活动记录打开详情。
3. 在「概览」中查看基础指标和轨迹缩略图。
4. 点击轨迹缩略图，可跳转到「轨迹分析工具」进行 3D 分析。
5. 切换到「复盘」查看疲劳、效率、事件和 AI 解读。

### 第四步：生成 AI 解读

1. 确认配置页网关连接正常。
2. 打开活动详情或复盘页。
3. 点击「AI 洞察」或「复盘 AI 解读」。
4. 等待系统生成分析结果。

如果是在轨迹分析工具中使用「活动建议」，可以先选择活动类型、填写计划活动时间，再点击「AI 生成建议」。未填写计划时间时，天气部分只会给出检查清单，不会推断具体天气。

## 四、典型使用场景

### 场景 1：导入一次跑步训练并复盘

导入 FIT 文件后，打开活动详情查看配速、心率、爬升和训练效果；再进入复盘页查看心率漂移、效率下降和疲劳区段，最后生成 AI 解读，了解本次训练的主要问题和改进方向。

### 场景 2：分析一条徒步或越野路线

在轨迹分析工具中导入 GPX 文件，查看距离、累计爬升、最高海拔和路线难度。需要出发建议时，填写计划活动时间和活动类型，生成补给、装备和体力安排建议。

### 场景 3：查看近期训练状态

在个人运动数据页同步最近活动后，查看雷达图和滚动指标，了解近期负荷、恢复、爬升能力和心率稳定性变化，再使用 AI 洞察辅助判断是否需要减量、恢复或调整训练重点。

### 场景 4：让 AI 助手回答训练问题

打开全局 AI 助手，可以询问近期跑量趋势、某次活动表现、半马备赛建议或恢复安排。AI 会基于脉图已有的运动上下文给出解释，但不会替代专业医疗或教练判断。

## 五、数据与隐私说明

脉图坚持本地优先原则。

- 脉图免费使用，本体在本地运行，不内置云端运动数据上传服务。
- 原始运动文件保存在本地。
- 核心数据库使用本地 SQLite。
- FIT 文件是活动事实源。
- AI 只消费经过后端整理的快照，不直接修改数据库事实。
- AI 输出是解释层内容，不会成为原始运动指标。
- AI 输出用于解释和参考，不等同于医疗建议、诊断结论或专业教练处方。
- 训练负荷、疲劳、恢复、补给和装备建议需要结合个人身体感受、环境风险和专业意见判断。
- 第三方平台账号授权由对应 skill 和平台处理；脉图不会要求你在应用内填写佳明或高驰账号密码。

使用外部大模型网关时，AI 请求会发送必要的摘要信息到你配置的接口。请根据自己的隐私要求选择可信的模型服务和网关。

## 六、常见问题

### 1. 为什么导入 GPX 后没有心率或功率？

GPX 主要保存路线点和轨迹信息，通常不包含完整的心率、功率、训练效果和跑步动态字段。需要完整运动数据时，请导入设备导出的 FIT 文件。

### 2. 如何批量下载 FIT 文件？

佳明大陆地区账户可以从 [Garmin 中国数据管理](https://www.garmin.cn/zh-CN/account/datamanagement/) 下载；佳明国际账户可以从 [Garmin 国际数据管理](https://www.garmin.com/en-US/account/datamanagement/) 下载；高驰用户可以从 [COROS 活动管理](https://t.coros.com/admin/views/activities) 下载。下载后，将 FIT 文件或压缩包通过「导入本地 FIT 文件」导入脉图。

### 3. AI 洞察一直加载或失败怎么办？

先进入「配置」页，检查大模型网关是否为绿色连接状态。若连接失败，请重新填写 API 地址、模型和 Token 后点击「测试连接」。

### 4. Garmin 同步按钮不可用怎么办？

请确认：

- 手表品牌已选择佳明。
- 已完成「告诉 AI 助手存储规范」。
- 已选择有效的开始日期和结束日期。
- 当前版本支持通过 OpenClaw 同步活动。

如果不是佳明用户，可以使用「导入本地 FIT 文件」。

### 5. 活动列表没有出现新导入记录怎么办？

可以尝试刷新活动列表，或确认 FIT 文件是否已成功解析。若文件重复，系统可能会跳过导入，避免生成重复活动。

### 6. 复盘页显示数据不足是什么意思？

部分复盘指标需要轨迹点、心率、距离、海拔、配速或功率等字段。如果活动是室内训练、短轨迹、缺少心率，或设备没有记录相关字段，复盘页会显示数据不足，而不是用前端或 AI 编造结果。

### 7. AI 建议可以作为安全预警吗？

不可以。活动建议不是安全预警、天气预报、医学判断或危险等级评估。它只是基于路线事实和你填写的计划信息生成准备建议，最终决策仍需结合实际天气、身体状态、装备和现场环境。

## 七、使用建议

- 想看完整运动指标，优先导入 FIT。
- 想看路线和爬升，可以导入 GPX。
- 每次同步或导入后，先在活动列表确认记录是否正确。
- 做训练复盘时，优先查看后端指标和曲线，再阅读 AI 总结。
- 使用活动建议前，最好手动填写活动类型和计划活动时间。
- 若 AI 输出与页面事实不一致，以页面后端指标和原始 FIT 数据为准。
