## 一、软件简介

脉图是一款本地优先的 AI 运动数据分析工具。它以 FIT 文件为核心数据来源，帮助用户导入、整理和复盘跑步、骑行、徒步、越野等运动记录，并通过可视化图表和 AI 解读，把原始运动数据转化为可理解的训练反馈。

脉图是一款免费、本地运行的运动数据分析工具，不会上传任何运动数据到云端。

脉图不是社交运动平台，也不是云端数据中台。它更像一个运行在本地的运动数据助手：负责保存可信运动档案、展示轨迹和指标、生成复盘洞察，并为 AI 助手提供稳定、可解释的运动上下文。

## 二、主要功能

### 1. 个人运动数据

个人运动数据页用于查看整体运动档案和历史活动记录。

主要能力包括：

- 查看 Garmin 用户画像、同步状态和基础运动信息。
- 按运动类型、标题关键词筛选历史活动。
- 分页浏览活动列表，查看距离、时长、爬升、心率、配速、功率等核心指标。
- 打开单条活动详情，进入概览或复盘分析。
- 删除不需要的运动记录。

### 2. FIT 数据导入与同步

脉图支持两种主要数据来源：

- 本地 FIT 文件：适合从手表、运动平台或备份目录导出的活动文件。
- Garmin 活动同步：佳明用户可在配置完成后按日期范围同步活动。

导入后，系统会解析 FIT 文件中的运动类型、时间、距离、轨迹、心率、配速、功率、海拔、圈数据等字段，并写入本地数据库。活动数据以本地 FIT 文件和后端解析结果为准，不依赖前端临时推算。

### 3. 活动详情

活动详情页用于快速了解单次运动。

你可以查看：

- 活动标题、时间、地点和运动类型。
- 距离、时长、爬升、消耗、平均心率、最大心率等核心指标。
- 真实比例轨迹缩略图。
- 圈数据、训练效果、环境挑战、路线难度等扩展指标。
- 轨迹缩略图入口，点击后可跳转到轨迹分析工具进行 3D 查看。

### 4. 运动复盘

复盘页面用于分析一次活动中身体状态和运动表现的变化。

它重点回答：

- 本次活动整体压力如何？
- 哪些阶段出现效率下降、心率漂移或疲劳累积？
- 是否存在能量断档、掉速、掉功率等线索？
- 哪些环境或路线因素影响了表现？
- 下一次类似活动应该如何调整节奏、补给或训练安排？

复盘数据来自后端权威快照，前端只负责展示，不自行补算事实字段。AI 洞察也只基于本次复盘快照生成，用于解释和建议，不会修改原始数据。

### 5. 轨迹分析工具

轨迹分析工具用于查看和分析 GPX/FIT 路线。

主要能力包括：

- 导入 GPX 轨迹文件。
- 在 3D 地球视图中查看路线。
- 查看海拔剖面、距离变化和路线统计。
- 在地图和剖面图之间联动定位。
- 导出 GPX 文件。
- 生成路线报告和活动建议。

对于 GPX 文件，脉图可以解析轨迹、距离、爬升和位置等路线信息；但 GPX 通常不包含完整的心率、功率、训练效果等设备字段。如需完整运动指标，建议优先使用 FIT 文件。

### 6. 雷达图与滚动指标

雷达图用于观察近期训练状态和关键能力维度。

系统会基于最近活动和滚动窗口计算指标，例如：

- 训练负荷趋势。
- HRV 或恢复相关基线。
- 心率漂移。
- 爬升能力。
- 阈值心率。
- 无氧峰值或强度表现。

雷达图 AI 洞察用于解释这些维度的变化，帮助你判断近期训练是偏恢复、偏积累，还是存在疲劳或强度失衡的迹象。

### 7. AI 功能

脉图的 AI 功能主要用于解释和建议，不作为事实来源。

常用入口包括：

- AI 洞察：针对单次活动或雷达图生成结构化解读。
- 复盘 AI 解读：基于本次运动复盘快照总结疲劳、效率、事件和建议。
- 活动建议：基于当前路线事实和用户手动填写的计划信息，生成补给、天气检查、装备和体力安排建议。
- 全局 AI 助手：根据当前运动画像和已导入数据回答训练相关问题。

AI 生成内容仅供训练参考。它不会改写 FIT 文件、不会写入原始指标，也不会把 AI 判断当作数据库事实。

### 8. 系统配置

配置页用于完成首次使用准备。

你可以在这里设置：

- 大模型连接方式：HTTP 网关或 CLI。
- HTTP 网关配置：服务商、API 地址、模型名称、Agent ID、Token 或鉴权信息。
- CLI 配置：CLI 类型、CLI 路径、OpenClaw Agent ID、CLI 模型、CLI 超时和 CLI 参数模板。
- 手表品牌。
- 本地 FIT 文件存储目录状态。
- 大模型连接测试。

HTTP 网关和 CLI 二选一即可，不需要同时配置两套大模型信息。配置完成后，脉图会在后续启动时进行静默验证。若配置页出现连接异常提示，需要根据当前连接方式检查网关 API 信息或本机 CLI 状态。

## 三、快速上手

### 第一步：完成基础配置

1. 打开脉图。
2. 进入「配置」页。
3. 选择大模型连接方式：HTTP 网关或 CLI。
4. 如果选择 HTTP 网关，填写 API 地址、模型、Agent ID 和 Token。
5. 如果选择 CLI，选择 CLI 类型；必要时填写 CLI 路径、OpenClaw Agent ID、CLI 模型和 CLI 超时。
6. 点击「测试连接」。
7. 选择手表品牌。
8. 根据设备品牌完成 Garmin 或 COROS 授权；如果你要使用按日期同步活动，请确认对应账号状态正常。

配置成功后，后续一般不需要重复操作。

### 大模型连接配置说明

脉图的大模型连接用于 AI 私教、活动建议、雷达图 AI 洞察和疲劳复盘 AI 洞察。它可以连接 HTTP 网关，也可以直接调用本机 CLI。

大模型连接现在支持两种方式：

- HTTP 网关：适合继续使用 OpenClaw / QClaw Gateway 或 OpenAI-compatible API 的用户。
- CLI：适合已经在本机配置好 OpenClaw CLI、Codex CLI、Claude Code 等命令行工具的用户。

两种方式二选一即可。选择 CLI 后，AI 私教、活动建议、雷达图 AI 洞察、疲劳复盘 AI 洞察等脉图内置 AI 功能都会统一走 CLI，不需要再额外配置 HTTP 网关。

注意：CLI 大模型通道只接管脉图内的 AI 对话与洞察生成。Garmin/COROS 账号授权、画像同步和活动同步走应用内置的账号同步能力，不会因为选择 CLI 而自动完成授权。

### 账号授权与数据同步说明

Garmin 和 COROS 同步能力随应用提供，普通用户不需要准备外部 skill 包。首次使用时，在「系统配置」里选择手表品牌和账号区域，然后点击「授权」或「检查状态」。

- Garmin：支持用户画像同步与按日期范围同步活动 FIT 文件。
- COROS：支持用户画像同步，并通过 COROS MCP 下载活动 FIT 文件。COROS MCP 的 FIT 下载单次最多返回 10 个活动；如果日期范围内活动较多，请缩小日期范围分批同步。
- 脉图不会在应用内询问或保存 Garmin / COROS 账号密码。正式安装包会随应用内置 COROS 授权所需的 Node.js 运行时，普通用户不需要单独配置依赖环境。

在「配置」页中，如果选择 HTTP 网关，OpenClaw 相关字段通常这样理解：

- 连接方式：选择「HTTP 网关」。
- 服务商：选择「本地 + MCP（OpenClaw / QClaw）」。
- 自定义 API 地址：填写 OpenClaw 提供的网关地址，例如本机服务地址或你实际部署的网关地址。
- 模型：选择或填写 OpenClaw 网关可调用的模型名称。
- Agent ID：如果 OpenClaw 给你分配了专用 Agent ID，就填入这里；没有则可以留空。
- Gateway Token：如果 OpenClaw 网关开启了鉴权，就填写 Token；如果你的本地网关没有鉴权，可以留空。

如果选择 CLI，相关字段通常这样理解：

- 连接方式：选择「CLI」。
- CLI 类型：选择 OpenClaw CLI、Codex CLI、Claude Code 或自定义 CLI。
- CLI 路径：可选。命令已经在系统 PATH 中时可以留空；OpenClaw CLI 在 macOS 上也支持留空后自动尝试使用 QClaw 自带路径。
- OpenClaw Agent ID：仅 OpenClaw CLI 显示。留空时使用 `main`；如果要进入指定 OpenClaw Agent，请填写类似 `agent-8b65944a` 的 ID。
- CLI 模型：可选。留空时由 CLI 自身默认配置决定。
- CLI 超时：默认 300 秒。模型响应慢时可以适当调大。
- CLI 参数模板：仅自定义 CLI 使用。路径栏只填写可执行文件路径，参数应填写到 CLI 参数模板中。

CLI 路径只填写可执行文件路径，不要填写 `codex exec`、`openclaw agent`、`claude -p` 这类参数。Codex、Claude Code 和 OpenClaw 的默认参数由脉图自动拼接；自定义 CLI 的参数写在「CLI 参数模板」里。

配置时建议按这个顺序操作：

1. 在脉图「配置」页选择手表品牌和账号区域。
2. 在脉图「配置」页选择 HTTP 网关或 CLI，并填写对应字段。
3. 点击「测试连接」。测试成功后，脉图会保存这组大模型配置。
4. 根据设备品牌点击「授权」或「检查状态」。如果你要使用按日期同步活动，请确认已完成对应品牌授权。

Garmin/COROS 账号同步依赖对应品牌的本机授权，不依赖 OpenClaw/LLM 提示词；同步得到的 FIT 文件会保存到应用当前固定本地目录。

### 第二步：导入运动数据

方式一：导入本地 FIT 文件

1. 进入「个人运动数据」页。
2. 点击「导入本地 FIT 文件」。
3. 选择 FIT 文件或 ZIP 压缩包。
4. 等待系统解析并刷新活动列表。

方式二：按日期同步账号活动

1. 确认手表品牌已选择为 Garmin 或 COROS。
2. 在配置页选择账号区域，并点击「检查状态」。如果尚未授权，按页面提示点击「授权」完成本机账号授权。
3. 在「个人运动数据」页选择开始日期和结束日期。
4. 点击「同步活动」。
5. 等待脉图下载 FIT 文件并导入本地活动记录，完成后活动列表会刷新。

COROS 活动同步目前按 COROS MCP 的单次下载限制处理，每次最多下载 10 个 FIT 文件；选择较长日期范围时，建议按几天或一周分批同步。COROS 官方同时对 FIT 文件请求设置每日限额，频繁重试时可稍后再试。

方式三：导入 GPX 轨迹

1. 进入「轨迹分析工具」页。
2. 点击「导入 GPX」。
3. 选择 GPX 文件。
4. 查看 3D 轨迹、海拔剖面和路线报告。

### 第三步：查看活动详情

1. 在「个人运动数据」页找到一条活动。
2. 点击活动记录打开详情。
3. 在「概览」中查看基础指标和轨迹缩略图。
4. 点击轨迹缩略图，可跳转到「轨迹分析工具」进行 3D 分析。
5. 切换到「复盘」查看疲劳、效率、事件和 AI 解读。

### 第四步：生成 AI 解读

1. 确认配置页大模型连接测试正常。
2. 打开活动详情或复盘页。
3. 点击「AI 洞察」或「复盘 AI 解读」。
4. 等待系统生成分析结果。

如果是在轨迹分析工具中使用「活动建议」，可以先选择活动类型、填写计划活动时间，再点击「AI 生成建议」。未填写计划时间时，天气部分只会给出检查清单，不会推断具体天气。

## 四、典型使用场景

### 场景 1：导入一次跑步训练并复盘

导入 FIT 文件后，打开活动详情查看配速、心率、爬升和训练效果；再进入复盘页查看心率漂移、效率下降和疲劳区段，最后生成 AI 解读，了解本次训练的主要问题和改进方向。

### 场景 2：分析一条徒步或越野路线

在轨迹分析工具中导入 GPX 文件，查看距离、累计爬升、最高海拔和路线难度。需要出发建议时，填写计划活动时间和活动类型，生成补给、装备和体力安排建议。

### 场景 3：查看近期训练状态

在个人运动数据页同步最近活动后，查看雷达图和滚动指标，了解近期负荷、恢复、爬升能力和心率稳定性变化，再使用 AI 洞察辅助判断是否需要减量、恢复或调整训练重点。

### 场景 4：让 AI 助手回答训练问题

打开全局 AI 助手，可以询问近期跑量趋势、某次活动表现、半马备赛建议或恢复安排。AI 会基于脉图已有的运动上下文给出解释，但不会替代专业医疗或教练判断。

## 五、数据与隐私说明

脉图坚持本地优先原则。

- 脉图免费使用，本体在本地运行，不内置云端运动数据上传服务。
- 原始运动文件保存在本地。
- 核心数据库使用本地 SQLite。
- FIT 文件是活动事实源。
- AI 只消费经过后端整理的快照，不直接修改数据库事实。
- AI 输出是解释层内容，不会成为原始运动指标。
- AI 输出用于解释和参考，不等同于医疗建议、诊断结论或专业教练处方。
- 训练负荷、疲劳、恢复、补给和装备建议需要结合个人身体感受、环境风险和专业意见判断。
- 第三方平台账号授权由应用内置授权入口和对应平台处理；脉图不会要求你在应用内填写佳明或高驰账号密码。

使用外部大模型网关时，AI 请求会发送必要的摘要信息到你配置的接口。使用 CLI 时，AI 请求会交给你本机配置的 CLI 工具处理。请根据自己的隐私要求选择可信的模型服务、网关或 CLI。

## 六、常见问题

### 1. 为什么导入 GPX 后没有心率或功率？

GPX 主要保存路线点和轨迹信息，通常不包含完整的心率、功率、训练效果和跑步动态字段。需要完整运动数据时，请导入设备导出的 FIT 文件。

### 2. 如何批量下载 FIT 文件？

佳明大陆地区账户可以从 [Garmin 中国数据管理](https://www.garmin.cn/zh-CN/account/datamanagement/) 下载；佳明国际账户可以从 [Garmin 国际数据管理](https://www.garmin.com/en-US/account/datamanagement/) 下载；高驰用户可以从 [COROS 活动管理](https://t.coros.com/admin/views/activities) 下载。下载后，将 FIT 文件或压缩包通过「导入本地 FIT 文件」导入脉图。

### 3. AI 洞察一直加载或失败怎么办？

先进入「配置」页，检查当前大模型连接方式是否测试成功。

- 如果使用 HTTP 网关，请检查 API 地址、模型、Agent ID 和 Token。
- 如果使用 CLI，请检查 CLI 类型、CLI 路径、OpenClaw Agent ID、CLI 登录状态，以及该命令是否能在系统终端中正常运行。

调整后重新点击「测试连接」。如果配置页提示「配置已修改，需重新测试」，说明当前表单已经不同于上次测试成功的配置，需要重新测试一次。

### 4. 账号同步按钮不可用怎么办？

请确认：

- 手表品牌已选择 Garmin 或 COROS。
- 已在配置页选择账号区域。
- 已完成对应品牌的本机授权；如果授权缺失，请回配置页点击「授权」。
- 已选择有效的开始日期和结束日期。
- 当前版本支持通过 Garmin provider 或 COROS MCP 同步活动 FIT 文件。

如果账号同步暂时不可用，可以使用「导入本地 FIT 文件」。

### 5. 活动列表没有出现新导入记录怎么办？

可以尝试刷新活动列表，或确认 FIT 文件是否已成功解析。若文件重复，系统可能会跳过导入，避免生成重复活动。

### 6. 复盘页显示数据不足是什么意思？

部分复盘指标需要轨迹点、心率、距离、海拔、配速或功率等字段。如果活动是室内训练、短轨迹、缺少心率，或设备没有记录相关字段，复盘页会显示数据不足，而不是用前端或 AI 编造结果。

### 7. AI 建议可以作为安全预警吗？

不可以。活动建议不是安全预警、天气预报、医学判断或危险等级评估。它只是基于路线事实和你填写的计划信息生成准备建议，最终决策仍需结合实际天气、身体状态、装备和现场环境。

## 七、使用建议

- 想看完整运动指标，优先导入 FIT。
- 想看路线和爬升，可以导入 GPX。
- 每次同步或导入后，先在活动列表确认记录是否正确。
- 做训练复盘时，优先查看后端指标和曲线，再阅读 AI 总结。
- 使用活动建议前，最好手动填写活动类型和计划活动时间。
- 若 AI 输出与页面事实不一致，以页面后端指标和原始 FIT 数据为准。
