# 脉图打包前必读

更新时间：2026-07-15

适用范围：Codex 或开发者执行脉图 macOS / Windows 打包、PyInstaller 构建、DMG/MSI 生成、发布前 smoke check。

## 0. Codex 打包任务硬规则

凡是用户要求“打包”“重新打包”“生成 DMG”“生成 MSI”“发布包”“PyInstaller 构建”，Codex 必须先阅读本文件，并把本文件视为打包前置契约。

除非用户明确要求只做只读分析，否则打包前必须完成本文的 Python 3.12 环境闸门。不得因为本机曾经通过、上次通过、旧 `.venv` 可用，跳过当前环境检查。

## 1. 必须在项目根目录执行

项目根目录：

```bash
/Users/fanglei/应用开发/AI track
```

打包前先进入项目根目录：

```bash
cd "/Users/fanglei/应用开发/AI track"
```

如果命令出现 `no such file or directory: scripts/...`，通常说明当前目录不是项目根目录；不要继续打包，先 `cd` 到项目根目录。

## 2. Python 3.12 环境闸门

脉图当前打包/验证环境以项目 `.venv312` 为准。禁止直接使用系统 `python3` 或旧 `.venv` 作为打包解释器。

每次打包前必须执行：

```bash
PYTHON=.venv312/bin/python scripts/install_packaging_deps.sh
.venv312/bin/python scripts/check_python312_runtime.py
.venv312/bin/python -m packaging_diagnostics
```

Windows 打包环境使用等价命令：

```powershell
.\scripts\install_packaging_deps.ps1 -Python .\.venv312\Scripts\python.exe
.\.venv312\Scripts\python.exe scripts\check_python312_runtime.py
.\.venv312\Scripts\python.exe -m packaging_diagnostics
```

## 3. 闸门必须检查什么

`scripts/check_python312_runtime.py` 必须通过，关键项包括：

- Python executable 指向项目 `.venv312`
- Python 版本为 3.12+
- `garminconnect == 0.3.6`
- `garmin-fit-sdk == 21.205.0`
- `curl_cffi >= 0.6`
- `fitparse / garmin_fit_sdk / garminconnect / curl_cffi / requests / pandas / numpy / scipy / webview / gpxpy / watchdog` 均可 import
- `import main` 后仍保持启动懒加载：`garmin_fit_sdk` 和 `fitparse` 不得被启动链路提前加载

`packaging_diagnostics` 已自动调用 runtime check。PyInstaller spec 中的 `check_packaging_prerequisites(os.getcwd())` 会在打包前阻断不合格环境。

## 4. 失败时怎么处理

如果出现以下错误，不得继续打包：

```text
Missing required distribution: garmin-fit-sdk
Cannot import garmin_fit_sdk
Python 3.12 runtime check failed
```

处理方式：

```bash
cd "/Users/fanglei/应用开发/AI track"
PYTHON=.venv312/bin/python scripts/install_packaging_deps.sh
.venv312/bin/python scripts/check_python312_runtime.py
.venv312/bin/python -m packaging_diagnostics
```

只有上述检查全部通过，才能继续 PyInstaller、DMG/MSI、签名、公证或发布包替换。

## 5. 边界

本闸门只负责打包环境和 Python 3.12 运行时依赖，不授权修改：

- FIT 解析算法
- `main.py` 同步业务逻辑
- `metrics_resolver.py` 指标算法
- `track.html` UI
- 数据库 schema / migration
- 用户真实数据

如果打包前置检查失败，优先修复环境或依赖声明；不要为了绕过打包闸门修改业务逻辑。
