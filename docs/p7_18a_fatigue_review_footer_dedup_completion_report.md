# P7.18A 复盘底部图层与摘要去重完成报告

## 任务目标

用户确认选择方案 A：底部保留为轻量“图表图例 / 交互栏 + 一行事件时间线”，删除与右侧摘要面板重复的底部结论卡片。

## 设计稿关系校正

- 设计草图的底部区域应辅助主图读图，而不是成为第二套右侧摘要。
- 当前右侧已经承担关键摘要、状态区间、建议等结论解释。
- 因此底部只保留图层状态、图层开关、事件锚点索引，避免“卡片套卡片”和信息重复。

## 契约约束

- 不启用 AI 洞察，不新增 `call_llm`，不修改 AI prompt。
- 不写 DB，不新增持久化字段。
- 底部只读后端 `get_fatigue_review(activity_id)` 白名单中的：
  - `data.curves`
  - `data.fatigue_zones`
  - `data.collapse_events`
- 底部不再读取或展示：
  - `data.metrics.training_load`
  - `data.metrics.hr_drift`
  - `data.metrics.decoupling`
  - `data.advice`
  - `data.disclaimer`
- 前端不得从 DOM、截图、ECharts 像素、活动标题、设备、路线或 `points` 推导运动事实。

## 实现内容

- 删除底部重复卡片：
  - 负荷
  - 状态
  - 建议状态
- 保留并轻量化：
  - 图层开关：曲线 / 疲劳带 / 事件 / Terrain
  - 聚合 chip：曲线、距离轴、疲劳带、事件
  - 一行事件时间线：最多展示 3 个 `collapse_events` 锚点
- CSS 将事件时间线从卡片网格改为横向 chip 条，避免抢占主图和右侧摘要。

## 验收结果

- 底部不再重复右侧关键摘要、状态区间、建议面板。
- 底部职责收敛为主图读图辅助。
- 右侧仍保留结论解释，信息架构更清晰。
