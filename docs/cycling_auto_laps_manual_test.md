# 骑行自动切圈 (V10.0) 手工测试清单 (P2-1)

> **版本**:2026-06-17
> **适用版本**:V10.0 全部任务(P0-1 / P0-2 / R-1~R-4 / P1-1 / P1-2)
> **对齐契约**:fit-arch-contrac §2.1(字段全链路可追溯)+ §2.2(数据可信分层)+ §八 8.3(Canonical DB 原则)
> **核心目的**:在自动化测试覆盖之外,补充真实 pywebview + 真实 DB + 真实活动数据下的端到端验收

---

## 0. 前置条件

### 0.1 环境

- [ ] App 正在运行(`python main.py` 已启动 pywebview)
- [ ] LLM 网关可连通(本地 OpenClaw / 远程 API 均可)
- [ ] 数据库 `~/.fitvault/user_profile.db` 存在且有数据
- [ ] 当前用户至少有 **23km 骑行活动**且 FIT 文件已导入

### 0.2 数据准备

V10.0 自动切圈仅作用于**户外骑行类**(cycling / road_cycling / mountain_biking),需要准备以下活动数据:

- [ ] **A 组专用**(骑行自动切圈):
  - [ ] ≥1 条 **23km 以上户外骑行** + **FIT 1 圈**(6/26 「雅安市 骑行」23.6km 可用)
  - [ ] ≥1 条 **23km 以上户外骑行** + **FIT 多圈**(≥2 圈)
  - [ ] ≥1 条 **5km 以下户外骑行**(6/17 「雅安市 骑行」4.0km 可用)
  - [ ] ≥1 条 **户外骑行 + 无 FIT 圈 + 无 points**(需手工构造)
  - [ ] ≥1 条 **户外骑行 + 无 FIT 圈 + 有 points**(需手工构造)

- [ ] **B/D 组专用**(圈表展示 / 其他运动回归):
  - [ ] ≥1 条 **跑步**活动(任意距离)
  - [ ] ≥1 条 **徒步**活动(任意距离)
  - [ ] ≥1 条 **室内骑行**(indoor_cycling)活动
  - [ ] ≥1 条 **游泳**活动

### 0.3 准备工具

- [ ] 打开 pywebview DevTools(`F12` 或右键 → Inspect)
- [ ] Console 标签页保留,准备看 `[V10.0 P1-1]` mock 过滤警告日志
- [ ] Network 标签页保留,准备看 `get_activity_detail` 响应中 `laps` 字段
- [ ] 准备截图工具,记录每个测试的最终结果
- [ ] 准备调整浏览器窗口大小的工具(响应式测试用)
