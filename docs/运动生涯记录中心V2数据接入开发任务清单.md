# 运动生涯记录中心 V2 数据接入开发任务清单

## 0. 使用说明

本文档是记录中心 V2 “数据接入”后续任务循环的顺序依据。它不替代既有 V2 UI/多运动交付手册，而是在视觉基本统一后，专门冻结从真实活动数据到记录中心展示数据的实现路径。

当前共识：

- 记录中心页面不从前端、图表或筛选项生成记录。
- `activities` 是真实运动事实源。
- Resolver 生成 `RecordEvidence`。
- 状态机决定 `active / candidate / ignored`。
- 前端只渲染 Records API ViewModel。
- 用户目前未授权写真实库；第一阶段只能做 dry-run preview。
- 用户目前未授权打包；本清单不包含打包任务。

## 0.1 数据接入总链路

```text
FIT / Garmin / COROS / 本地导入
        ↓
activities 真实活动表
        ↓
Activity Adapter 标准化事实
        ↓
Record Resolver 生成 RecordEvidence
        ↓
Record State Machine 判断 active / candidate / ignored
        ↓
career_pb_records / career_event_candidates
        ↓
Records API ViewModel
        ↓
前端记录中心展示
```

当前缺口是：

```text
activities → Activity Adapter → RecordEvidence → 状态机 / dry-run preview
```

## 0.2 全局硬约束

所有任务必须遵守：

- 第一阶段不得写真实库。
- 不得打包。
- 不得提交 Git，除非用户明确要求。
- 不得新增与 `career_pb_records`、`career_record_events`、`career_event_candidates` 语义重叠的事实表。
- 不得从前端计算记录事实、窗口、Scope、提升、置信度。
- 不得把 mock 数据渲染成真实记录。
- 不得暴露 raw FIT、完整轨迹、原始功率流、GPS 点、文件路径、SQLite schema、设备序列号、账号 token、体重历史。
- 所有记录证据必须绑定 `activity_id`。
- 所有窗口类记录必须绑定 Activity 内范围。
- `validation_required` 不得自动进入 active。
- `candidate_only` 不得自动进入 active。
- `analysis_only / model_only / unavailable` 不得进入状态机。
- 用户确认候选时只能确认后端原始 evidence，不得提交修改后的成绩、距离、时间、功率、海拔、scope、range 或 reason。

## 0.3 标准距离统一规则

所有“标准距离成绩”统一采用 Best Effort Distance：

```text
标准距离 PB = 所有活动中，该标准距离连续窗口的最短 elapsed time
```

不是：

```text
整次活动距离刚好接近标准距离才算 PB
```

也不是：

```text
用整次活动均速倒推标准距离成绩
```

适用范围：

- 跑步：5K、10K、半程马拉松、马拉松。
- 骑行：10K、20K、40K、50K、100K、180K。
- 泳池游泳：50m、100m、200m、400m、800m、1500m，使用 lap/length 序列。
- 公开水域：750m、1500m、1900m、3800m、5K、10K，使用 GPS/距离时间流且质量门禁更严。

缺少可靠距离-时间序列时，只能 fallback 到整次活动匹配，并必须标记：

- `fallback_activity_total`
- lower confidence
- candidate 或 validation_required

不得直接进入 active。

## 0.4 暂不纳入范围

以下能力暂不进入当前记录中心：

- 同赛道最好成绩。
- Route PR。
- Segment PR。
- 爬坡赛段 PR。
- 任何依赖赛道管理、路线实体、路线签名、同向/反向判断、路线重合度和用户路线校正的记录。

原因：当前产品还没有赛道管理能力。没有赛道实体和用户校正入口时，这类记录不可解释，不能进入正式记录中心。

未来等赛道管理模块成熟后，再作为独立能力接入。

## 0.6 阶段门禁

### 阶段 A：dry-run preview，只读预览

目标：真实 `activities` 进入 Resolver，但不写库。

允许：

- 读取 `activities`。
- 生成安全 preview ViewModel。
- 返回每个 record_key 的候选结果、窗口、质量和 reason。

禁止：

- 写 `career_pb_records`。
- 写 `career_event_candidates`。
- 写 `career_record_events`。
- 修改 active 记录。

### 阶段 B：写候选，不写 active

必须等待用户明确授权。

允许：

- 把 dry-run 中可信的 evidence 写入 `career_event_candidates`。
- 所有 available 记录也可以先降级候选。

禁止：

- 自动写 active。
- 自动替换旧记录。

### 阶段 C：正式 active 写入

必须等待用户明确授权。

允许：

- 高置信 available 记录进入 `career_pb_records active`。
- 原 active 进入 superseded。
- 写事件日志。

禁止：

- `validation_required` 自动 active。
- `candidate_only` 自动 active。
- `model_only / analysis_only / unavailable` 进入正式记录。

  distance_m
  ascent_m
  max_altitude_m
  power_points_available
  distance_time_stream_available

  raw_quality_flags

内部可使用：

- `points_json / track_json`

   - `activity_missing`

   - `mock_activity_excluded`

   - `power_stream_missing`
   - `lap_length_stream_missing`
   - `ebike_scope_excluded`

## RCDI-04：通用 Best Effort Distance Resolver 核心

### 任务目标

实现跨运动通用的连续标准距离窗口 Resolver。

### 规则

```text
best_effort_distance(distance_m) =
  min(elapsed_time_sec over contiguous distance-time windows)
```

### 输入

- 标准化 distance-time stream。
- 标准距离列表。
- sport quality policy。

### 输出

`RecordEvidence` 或 dry-run evidence dict：

- `record_key`
- `activity_id`
- `metric.elapsed_time_sec`
- `range.start_sec`
- `range.end_sec`
- `range.start_distance_m`
- `range.end_distance_m`
- `quality.confidence`
- `quality.reason_codes`

### 质量门禁

- 使用 elapsed time。
- 窗口不得跨无效时间戳。
- 不得用 moving time。
- 不得用整次均速倒推。
- stream 缺失时返回 fallback/ignored，不直接 active。

### 验收标准

- 单测覆盖：
  - 15km 内最快 5km。
  - 5.6km 内最快 5km 优于 15km 内 5km。
  - stream 缺失 fallback。
  - 时间断点。
  - 距离回退/跳点。

- 不得调用真实 apply。
- 不得生成 active。

- 数据库计数不变。
- 性能有 batch/max_activities/cancel_after 控制。
