# 趋势分析线框图 v0.1

> **状态**: 初稿  
> **日期**: 2026-06-21  
> **关联文档**: `docs/trend_analysis_prd_v0_1.md`  
> **目标**: 定义「趋势分析」模块的信息布局、页面结构和关键交互,不定义最终视觉风格

---

## 1. 页面入口

建议在主导航或运动复盘系统中新增一级入口:

```text
┌────────────────────────────────────────────────────────────────────────────┐
│ 脉图                                                                       │
├──────────────┬─────────────────────────────────────────────────────────────┤
│ 轨迹库        │  当前页面标题                                               │
│ 活动详情      │                                                             │
│ 用户画像      │                                                             │
│ 趋势分析  ●   │                                                             │
│ 设置          │                                                             │
└──────────────┴─────────────────────────────────────────────────────────────┘
```

入口原则:

- 「趋势分析」是跨活动模块,不应只放在单次活动详情内。
- 单次活动详情可提供跳转入口,例如「查看长期趋势」。

### 2.2 总览页组件说明

| 区域 | 组件 | 目的 |
|---|---|---|
| 顶部标题 | 标题 + 范围摘要 | 明确当前分析口径 |
| 筛选条 | 运动、时间、口径、刷新 | 控制分析范围 |
| 四张状态卡 | 能力、负荷、恢复、效率 | 先给结论 |
| 负荷趋势 | ATL/CTL/TSB 组合图 | 解释负荷状态 |
| 强度分布 | 堆叠柱状图 | 解释训练结构 |
| 能力表现 | 指标摘要 | 连接能力详情 |
| 恢复与效率 | 指标摘要 | 连接恢复详情 |

## 9. 导航与交互

### 9.1 页面关系

```text
趋势分析总览
  ├─ 训练负荷详情
  ├─ 能力表现详情
  ├─ 恢复与效率详情
  └─ 复合指标抽屉
```

### 9.2 关键交互

| 交互 | 行为 |
|---|---|
| 切换运动类型 | 重新计算并刷新所有趋势 |
| 切换时间范围 | 重新计算窗口内指标 |
| 点击总览卡 | 打开对应详情或解释抽屉 |
| 点击可信度 | 展示数据质量和降级原因 |
| 点击图表点 | 展示对应日期/周和纳入活动 |
| 点击证据活动 | 跳转到活动详情 |

## 10. UI 优先级

### M0 必做

- 趋势总览页。
- 4 张状态卡。
- 训练负荷组合图。
- 强度分布堆叠柱。
- 数据不足空状态。
- 可信度展示。

### M1 必做

- 能力表现详情页。
- 最佳努力曲线。
- 同心率配速趋势。
- 证据活动表。

### M2 必做

- 恢复与效率详情页。
- HRV / 静息心率图。
- 心率漂移卡。
- 影响因素标签。

### M3 必做

- 复合指标展开抽屉。
- AI 解释层入口。
- 个性化建议摘要。

---

## 11. 设计约束

- 不把所有模型都做成折线图。
- 总览页先给判断,再给证据。
- 每个趋势判断必须展示可信度。
- 数据不足时不画趋势。
- 复合指标必须可展开查看贡献项。
- 前端只渲染后端输出,不计算核心运动生理指标。
