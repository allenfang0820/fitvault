# P7.17B 复盘底部视觉减重与图表宽度校正完成报告

## 任务目标

对照设计草图，继续回正 P7.17 后底部区域过重、工程说明过多的问题，并排查用户截图中“X 轴右侧大片留白”的异常来源。

## 设计稿关系校正

- 设计稿中底部信息是主图后的轻量辅助层，不应抢占“多维时间轴分析”主图的视觉重心。
- 本次将底部从四块较重的说明卡和多枚明细 chip，收敛为紧凑图层栏、聚合数据状态和轻量摘要。
- 保留事件、负荷、状态、建议四类信息，但标题和密度降级，避免变成工程验收面板。

## 契约约束

- 不启用 AI 洞察，不新增 `call_llm` 调用，不修改 AI prompt。
- 不写 DB，不新增持久化字段。
- 底部摘要只读 `get_fatigue_review(activity_id)` 后端快照白名单：
  - `data.curves`
  - `data.collapse_events`
  - `data.fatigue_zones`
  - `data.metrics`
  - `data.advice`
  - `data.disclaimer`
- 前端不得从 DOM、截图、ECharts 像素、活动标题、设备、路线或 `points` 推导运动事实。
- 新增的图表宽度校正只读取容器尺寸并调用 ECharts `resize()`，属于布局修正，不生成运动结论。

## 实现内容

### 1. 底部视觉减重

- `fr-chart-footer` 增加 `fr-chart-footer-compact`。
- 标题从“图层与底部分析”收敛为“图层与摘要”。
- 边界说明降低透明度和字号，仅作为契约痕迹存在。
- chip 从逐曲线明细改为聚合状态：
  - 曲线可用数 / 总数
  - 距离轴点数
  - 疲劳带数量
  - 事件数量
- 底部卡片标题从“负荷与能量 / 状态解释”收敛为“负荷 / 状态”。

### 2. 图表宽度稳定校正

- 新增 `_resizeFatigueReviewChartWhenStable()`。
- 在以下场景触发：
  - 首次渲染复盘主图后。
  - 分层 ECharts `setOption()` 后。
  - 切换复盘 Tab 后。
  - 图层开关切换后。
- 使用即时、`requestAnimationFrame`、80ms、220ms 多次短窗口校正，降低 display:none 切换、底部渲染、右栏布局变化造成的宽度塌陷概率。

## X 轴异常排查结论

已对用户截图对应活动 `2026-06-03 西城区 跑步` 的后端快照做只读排查：

- 活动总距离约 `6.015 km`。
- `curves.distance` 为 `2527` 点。
- `curves.distance` 最大值约 `6.103 km`。
- `hr / speed / altitude / terrain_load / efficiency / gap / grade / time` 均为 `2527` 点。

因此，这次右侧大片留白不属于后端距离轴过短；更符合前端 ECharts 在复盘 Tab / 两栏布局 / 底部渲染后没有稳定 resize 的表现。本次已按这个方向修正。

## 验收建议

1. 打开 `2026-06-03 西城区 跑步` 复盘页。
2. 确认分层主图横向铺满右侧画布，X 轴 0-6km 范围内曲线不再集中在左侧。

4. 切换详情 Tab 后再回到复盘，确认图表宽度正常。
5. 对照设计草图，确认底部只作为轻量辅助信息，不压过主图。
