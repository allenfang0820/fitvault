# P7.18D 复盘泳道顺序、Tooltip 与左侧标签回正完成报告

## 任务目标

根据用户截图反馈，回正复盘分层主图的读图体验：

- 光标旁 tooltip 数值最多保留一位小数，整数直接显示整数。
- tooltip 指标顺序必须与泳道自上而下一致。
- 泳道顺序调整为：心率、配速、GAP、效率、海拔、坡度、Terrain Load。
- 左侧泳道标题取消方框，居中对齐泳道。
- 泳道标题翻译为中文。

## 设计稿关系校正

- 设计草图强调分层泳道的顺序阅读，tooltip 也必须服从同一顺序。
- 左侧标题应作为泳道标签，而不是独立卡片按钮。
- Terrain Load 在 UI 上翻译为“地形负荷”，与其他中文标题一致。

## 契约约束

- 不启用 AI 洞察，不新增 `call_llm`，不修改 AI prompt。
- 不写 DB，不新增持久化字段。
- 不从 DOM、截图、ECharts 像素、活动标题、设备、路线或 `points` 推导运动事实。
- 前端仅调整分层主图的展示顺序、标签文案、tooltip 格式和可见性控制。
- 后端字段仍保持 `curves.speed`、`curves.terrain_load` 等契约字段不变。

## 实现内容

- 新增 `_frFormatTooltipNumber(value)`：
  - 数值四舍五入到一位小数。
  - 一位小数为 `.0` 时显示整数。
- ECharts tooltip 自定义 formatter：
  - 按 `lanes` 顺序组织指标。
  - 显示中文指标名。
  - 使用统一数值格式。
- 泳道顺序调整为：
  1. 心率
  2. 配速
  3. GAP
  4. 效率
  5. 海拔
  6. 坡度
  7. 地形负荷
- 左侧泳道标签：
  - 删除编号方块。
  - 删除卡片边框和背景。
  - 标题居中对齐泳道。
- 图层控制条同步调整：
  - “速度”显示为“配速”。
  - “Terrain Load”显示为“地形负荷”。
  - 控制顺序与泳道一致。

## 验收结果

- tooltip 顺序与泳道顺序一致。
- tooltip 数值不再出现过长小数。
- 左侧标签不再像按钮/卡片，回归泳道标签语义。
- 泳道标题中文化，整体读图更直接。
