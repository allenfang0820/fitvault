# P7.18B 复盘底部事件时间线删除完成报告

## 任务目标

用户确认：底部“事件时间线”和状态阶段概览、主图图钉存在职责重叠，要求删除事件时间线及其所在整个卡片区域。

## 设计稿关系校正

- 主图中的图钉 / 竖向参考线已经表达关键事件点。
- 状态阶段概览和右侧状态区间已经表达疲劳区间。
- 底部事件时间线在当前数据形态下只是重复索引，不再保留。
- 底部最终收敛为图层控制与数据可用性状态，服务读图，不输出结论。

## 契约约束

- 不启用 AI 洞察，不新增 `call_llm`，不修改 AI prompt。
- 不写 DB，不新增持久化字段。
- 底部 footer 不再读取 `data.collapse_events`。
- 事件信息继续由主图图钉 / 竖向参考线和右侧面板承载。
- 前端不得从 DOM、截图、ECharts 像素、活动标题、设备、路线或 `points` 推导运动事实。

## 实现内容

- 删除 `fr-footer-event-strip` 整个底部事件时间线区域。
- 删除 `fr-footer-event-timeline` DOM。
- 删除底部时间线渲染逻辑。
- 删除已无用途的底部事件 chip 样式。
- 更新质量门测试：
  - footer 必须保留图层控制和聚合 chip。
  - footer helper 不得读取 `data.collapse_events`。
  - footer 区域不得出现事件时间线、负荷、状态、建议等重复卡片。

## 验收结果

- 底部只剩轻量图层控制区域。
- 事件职责回归主图图钉和右侧摘要。
- 信息架构更清晰，底部不再与状态阶段概览重复。
